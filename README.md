# 选课系统（Qt/C++）

大一暑期课程项目：基于 Qt Widgets 的 GUI 选课系统，使用 SQLite 存储用户与课程数据，实现登录、选课、退课与课程管理的完整流程，强调数据库建模与业务规则落地。

## 项目亮点
- Qt Widgets + C++17：桌面端 GUI，交互直观、流程清晰。
- 轻量数据库：Qt SQL + SQLite，本地文件即用即跑。
- 选课业务闭环：开放/关闭选课、容量控制、选课与退课。
- 关系数据建模：`userinfo`、`cusinfo`、`enrollment` 等表，包含外键与唯一约束。

## 功能概览
- 登录与角色：学生 / 教务处 / 辅导员三类入口。
- 学生端：课程列表、已选课程、选课/退课、个人信息查看。
- 管理端：课程新增/删除、课程信息查看、系统开放/关闭选课。
- 用户管理：学号注册、查询、修改与删除。

## 技术栈
- 语言：C++17
- GUI：Qt Widgets
- 数据库：SQLite（Qt SQL 模块）
- 构建：qmake（`.pro` 工程）

## 目录结构
- `*.ui`：Qt Designer 界面文件
- `*.cpp/*.h`：业务逻辑与界面交互
- `Tutorial.pro`：Qt 工程配置

## 数据库设计
应用启动时会自动创建表（若不存在），默认数据库文件名为 `hsw.db`：
- `userinfo`：用户账号与基础信息
- `cusinfo`：课程信息（容量与已选人数）
- `enrollment`：选课关系（带外键与唯一约束）
- `system_status`：选课系统开关状态

> 数据库文件会生成在运行目录，建议不要提交到 GitHub。

## 快速开始
1. 使用 Qt Creator 打开 `Tutorial.pro`。
2. 配置 Qt Kit（建议 Qt 5.15+ 或 Qt 6）。
3. 构建并运行，首次启动会自动初始化数据库。
